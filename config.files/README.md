# Zoho Analytics Design System — AI Pipeline

> **Figma Source:** [Design System – Zoho Analytics](https://www.figma.com/design/m2iOWX3I9aDI5kgyw4wCo0/Design-System--Zoho-Analytics)  
> **Stack:** Claude + Figma MCP (Desktop Bridge Plugin)  
> **Purpose:** AI-driven pipeline to create pixel-perfect Figma components from design briefs, fully token-bound, zero hardcoded values.

---

## ⛔ Prime Directive

> **The only valid output of this pipeline is a published Figma component.**

| Input | What must happen | What must NEVER happen |
|---|---|---|
| Screenshot(s) | Build the component in Figma | Output .md or .json as a substitute |
| Figma link | Build/update in Figma | Treat "reading the link" as done |
| Description | Build in Figma | Output a spec JSON and stop |
| No Bridge connection | Output PIPELINE BLOCKED | Generate any files as fallback |

`.md` docs and `.json` configs are only created in **STEP 12**, after the component is built AND published in Figma. They are a byproduct — never the deliverable.

---

## Quick Start for Team Members

1. Open **`pipeline/PIPELINE-BRIEF.md`**
2. Fill in the component name (and optional Figma URL / notes)
3. Drag the file into your Claude chat
4. Attach screenshots to the same chat message
5. Type `"build"` → agent runs the full pipeline automatically

> The agent will handle everything: reading the codebase, extracting tokens, building in Figma, verifying against your screenshots, and publishing.

## Pipeline at a Glance

```
Input: text brief / screenshot(s) / Figma link
        │
        ▼
  [00] Code Agent ────────── reads codebase/ (READ ONLY)
        │                     → exact variantAxes, booleanProperties,
        │                       textProperties, measurements (px)
        ▼
  [01] Spec Agent ─────────── validates brief → component spec JSON
        │                     Code Agent output = highest-priority input
        ▼
  [02] Token Resolver ──────── resolves token dot-paths → hex/px/Figma RGB
        │                     uses Code Agent exact px — no estimation
        ▼
  [03] Vision Agent ───────── analyzes screenshots (optional)
        │                     2+ screenshots → side-by-side comparison table
        │                     → what DIFFERS = variant axis / boolean / text
        │                     → what is SAME = shared property
        │                     Code Agent ran? → colors/shadows only (skip structure)
        ▼
  [04] Orchestrator ───────── STEP 0: Figma Bridge check (hard block if fails)
                              STEP 1–12: validate → search → plan → confirm
                              → proceed → build → audit → verify → publish → files
```

**Priority for all property decisions: Code Agent > Figma URL > Vision Agent**

## Key Rules (all agents enforce these)

### 1 — Search before creating
Read `docs/COMPONENT-REGISTRY.md` first.
- `✅ Done` → find and reuse — **never duplicate**
- `🔄 Partial` → find in Figma, build only the missing piece
- `⏳ Pending` → search library + local pages — build only if all return nothing

### 2 — Multi-screenshot = one COMPONENT_SET
2 or more screenshots of the same component → always **1 COMPONENT_SET**.

Vision Agent compares all screenshots in a side-by-side table:
- Property same across all → **shared property**
- Property differs between screenshots → **variant axis**
- Element appears / disappears → **boolean property**
- Text content differs → **text property**

Orchestrator outputs a count verification: `N axes × M values = total variants` before building.

### 3 — Icon and image placeholders only
Never import SVG icons or image assets. Always use:
```js
// Icon
const iconSlot = figma.createFrame();
iconSlot.name = 'icon--{purpose}';  // icon--chevron, icon--search, icon--left
iconSlot.resize(16, 16);
iconSlot.fills = [];
iconSlot.strokes = [{ type: 'SOLID', color: { r: 0.5, g: 0.5, b: 0.5 } }];
iconSlot.strokeWeight = 1;
iconSlot.dashPattern = [2, 2];

// Image
const imgSlot = figma.createFrame();
imgSlot.name = 'image--placeholder';
imgSlot.fills = [{ type: 'SOLID', color: { r: 0.878, g: 0.898, b: 0.922 } }];
```

### 4 — Screenshot fidelity check
Vision Agent must output a **FIDELITY CHECK** table before handing off:
- Each color: observed hex → nearest token → RGB delta (ΔTotal)
- ΔTotal ≤ 15 → confident match · ΔTotal > 30 → UNCERTAIN, ask user
- Low-confidence measurements → use Code Agent value or ask user
- No UNCERTAIN values may be unresolved before building starts

### 5 — Token-bound values only
Every fill, stroke, radius, padding, gap, and font size must bind to a Figma library variable via `setBoundVariableForPaint()`. Zero hardcoded hex or px. Token audit (STEP 8) enforces this — any FAIL blocks publishing.

### 6 — Confirm before executing
The Orchestrator outputs the full plan at STEP 5 and waits for `"proceed"` before any `figma_execute` write call.
- Known components → compact FAST CONFIRM format
- New/unknown components → full plan with all sections

## Quality Gates

| Gate | Tool | Pass criteria | When |
|---|---|---|---|
| **G2** Token Audit | `scripts/token-audit.js` | 0 FAIL (0 hardcoded values) | STEP 8 — before screenshot |
| **G1** Visual Diff | `config/verify-rubric.json` | Score ≥ 14/16 per variant | STEP 9 — after token audit |
| **G3** A11y Check | Inline `figma_execute` | Contrast ≥ 4.5:1, touch targets ≥ 44px, all layers named | STEP 10 — after visual diff |

Multi-screenshot builds: G1 is checked **per variant** against its reference screenshot (not just one overall screenshot of the full set).

---

## Fix Loop

Steps 7 → 8 → 9 = one iteration. Maximum 3 iterations.

```
Attempt 1 → FAIL → apply fixActions for failed checks only → re-verify
Attempt 2 → FAIL → same check failed again? → PARTIAL ESCALATE that check
                   new failures? → apply fixActions for new failures only
Attempt 3 → FAIL → ESCALATE TO HUMAN — all MCP execution stops
```

- Fixes are **surgical** — only touch the failing checks, never rebuild passing ones
- C10 (naming only) → fix inline, no iteration consumed
- Same check failing twice → escalate that check, continue for the rest
